Hilfe

In diesem Editor schreibst du deinen Bulibot als Groovy-Skript. Du kannst mehrere Versionen verwalten und es gibt immer genau eine Live-Version. Diese wird mit dem unten aufgeführten Kontext pro anstehendem Spiel einmal aufgerufen um das Tippergebnis zu berechnen. Ebenso kann jede beliebige Version gegen zur Verfügung stehende Testdaten ausgeführt werden. Die Testdaten bestehen immer aus bereits komplett abgeschlossenen Spieltagen.

Idee

Die Idee besteht darin, alle Ergebnisse aufgrund von statistischen Daten zu berechnen. Hierzu dienen die Klassen Statistics und StatisticsResult. Mit Hilfe von Statistics können über einen StatisticsBuilder neue StatisticResult Objekte erzeugt werden, der StatisticsBuilder bietet hierfür eine Menge an Filter-Methoden. Ein StatisticsResult basiert immer auf Grundlage einer Liste von Spielen, diese werden im StatisticsBuilder gefiltert. Das StatisticsResult enthält daher einige allgemeine Daten, sowie eine Tabelle und Torschützen Daten auf Grundlage der gefilterten Spiele. Leider sind die OpenligaDB-Daten zu den Torschützen aktuell noch nicht zuverlässig gepflegt.

Kontext

Die folgenden Variablen sind bei jedem Aufruf im Kontext des Groovy-Skriptes verfügbar:

Variable Typ Beschreibung
match MatchMetadata Metadaten des zu tippenden Spiels.
statistics Statistics Allgemeine Statistiken.
seasonData StatisticsResult Vorberechnete Statistik Ergebnisse für die aktuell laufende Saison.
lastSeasonData StatisticsResult Vorberechnete Statistik Ergebnisse für die vergangene Saison.
overallData StatisticsResult Vorberechnete Statistik Ergebnisse für alle im System verfügbaren Daten.

Für den Fall einer Ausführung mit Testdaten stehen im Ergebnis einige Analyse-Möglichkeiten zur Verfügung. Zum einen werden Ausgaben auf Stdout (mittels println) als Debug-Log zur Verfügung gestellt, zum anderen können die im Skript definierten Variablen und ihre Werten eingesehen werden. Aufgrund der Datenmengen werden die oben aufgelisteten initialen Kontextvariablen jedoch nicht mit aufgeführt.

Rückgabe

Die Rückgabe des berechnetes Tipp-Ergebnisses erfolgt dadurch, dass im Groovy-Skript zwei Variablen mit den Toren des Heim- bzw. Gast-Teams gesetzt werden. Diese werden nach der Ausführung entsprechend ausgewertet.

Variable Beschreibung
goalsTeamOne Tore des Heim-Teams.
goalsTeamTwo Tore des Gast-Teams.

Fehlt eine dieser Variablen, besitzt keinen Wert oder kommt es zu einem Laufzeitfehler, so wird die Berechnung als fehlgeschlagen angesehen.
